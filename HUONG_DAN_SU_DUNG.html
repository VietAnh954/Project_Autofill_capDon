3. Cách khởi động Dashboard
Làm mỗi ngày khi bắt đầu làm việc
  • 1
    Mở Outlook Desktop

    Đảm bảo Outlook đang chạy và đã đăng nhập vào anh.nguyen1@affina.com.vn. Đợi Outlook đồng bộ mail mới (~1 phút).

  • 2
    Mở PowerShell (hoặc Terminal)

    Nhấn Windows + X → chọn Terminal hoặc Windows PowerShell.

  • 3
    Di chuyển vào thư mục project
    cd "C:\Users\ADMIN\Documents\Work\Project_Autofill_capDon"
  • 4
    Khởi động server Dashboard
    python -m auto_fill serve
    INFO: Started server process
    INFO: Waiting for application startup.

    Khi thấy dòng xanh "Uvicorn running" là server đã sẵn sàng. Không đóng Terminal này.

    Dashboard sẽ hiện ra với giao diện tương thích mọi kích cỡ cửa sổ. Để tiện, bạn có thể bookmark trang này.

▶ Nút "Chạy ngay" — Dùng hàng ngày

Bắt đầu toàn bộ quy trình: phân loại mail → tải file → chuẩn hóa → kiểm tra trùng → ghi Excel tổng → chuyển mail vào đúng folder.

🔍 Nút "Dry Run" — Dùng để kiểm tra

Chạy thử — đọc mail và phân tích file, KHÔNG ghi gì vào Excel tổng và KHÔNG chuyển mail. Dùng để xem trước kết quả trước khi chạy thật.

🔄 Nút "Làm mới"

Tải lại số liệu thống kê từ database. Không chạy pipeline.

5. Quy trình sử dụng hàng ngày
Làm theo đúng thứ tự này mỗi ngày
  • 1
    Mở Outlook — đợi mail đồng bộ xong

    Đợi Outlook tải mail mới từ server (~1 phút). Bạn sẽ thấy các mail mới trong Inbox.

  • 2
    Mở Dashboard (nếu chưa mở)

    Chạy python -m auto_fill serve trong Terminal → mở trình duyệt → http://localhost:8080/dashboard

  • 3
    (Tùy chọn) Chạy Dry Run để kiểm tra trước

    Nhấn 🔍 Dry Run → xem Log Console — hệ thống liệt kê tất cả hồ sơ tìm thấy mà không ghi gì.

  • 4
    Nhấn "Chạy ngay" để xử lý thật

    Nhấn ▶ Chạy ngay → badge đổi sang Đang chạy → theo dõi Log Console bên dưới.

  • 5
    Đợi hoàn thành và kiểm tra kết quả

    Khi badge đổi sang Hoàn thành: xem các thẻ số đã tăng chưa. Kiểm tra Outlook — mail đã xử lý sẽ nằm trong các folder tương ứng (xem mục 9).

  • 6
    Kiểm tra folder Needs_Review (nếu có)

    Mail cần xem lại tay sẽ vào Inbox → AutoFill → Needs_Review. Xem chi tiết ở mục 9.

✅

Bình thường: Nếu không có mail mới phù hợp, pipeline hoàn thành ngay và log hiện [done] processed=0 written=0 — không phải lỗi, chỉ là không có việc để làm.

6. Đọc Log Console — hiểu kết quả
Biết pipeline đang làm gì
[run] dry_run=False source=outlook
→ Pipeline bắt đầu, chạy thật, nguồn Outlook

[mail_type] subject="Affina Care tái tục..." → TYPE B (GCN PDF)
→ Hệ thống nhận diện loại mail

[mail_type] subject="CHECK PHƯƠNG ÁN..." → TYPE C → Needs_Review
→ Mail check phương án, chuyển vào Needs_Review

[ok] Nguyễn Văn A — Sức khỏe → ghi dòng #128
→ Hồ sơ được ghi vào Excel tổng thành công

[dedup] Bỏ qua: 012345678901 — Nguyễn Thị B (đã tồn tại)
→ Hồ sơ trùng với dữ liệu cũ, bỏ qua, KHÔNG ghi đè

[done] processed=5 written=3 duplicates=1 review=1 skipped=0 errors=0
→ Tổng kết: 5 mail xử lý, 3 ghi thành công, 1 trùng, 1 cần review

Các trường hợp cần chú ý

Dòng logÝ nghĩaHành động
[ok] Ghi thành công vào Excel tổng Không cần làm gì
[dedup] Bỏ qua Hồ sơ đã có trong file tổng, bỏ qua để tránh trùng Bình thường. Nếu cần cập nhật → sửa tay trong Excel
[review] Mail cần xem lại tay (CHECK phương án, thiếu field...) Kiểm tra folder Needs_Review trong Outlook
[skip] Mail báo cáo/thông báo, không cần xử lý Bình thường, hệ thống tự bỏ qua
[error] Lỗi (Outlook không kết nối, file hỏng...) Xem chi tiết log, kiểm tra Outlook đang mở không

8. Hệ thống nhận diện 5 loại mail
Tự động phân loại trước khi xử lý — không nhầm mail báo cáo

Từ v0.7.0, mỗi mail được phân loại thành 1 trong 5 TYPE trước khi quyết định có ghi vào Excel tổng hay không.

TYPE A
Cấp đơn mới (Excel)
Mail có tiêu đề "Cấp đơn mới" / "Phiếu cấp đơn" + đính kèm file Excel. Đây là loại phổ biến nhất từ Affina (mail2 mẫu).
✅ Tự động ghi vào Excel tổng
TYPE B
GCN / Tái tục (PDF)
Mail kèm file PDF Giấy Chứng Nhận hoặc thông báo tái tục bảo hiểm. Hệ thống đọc và trích xuất thông tin từ PDF.
✅ Tự động ghi vào Excel tổng
TYPE C
Check phương án / Hỏi giá
Mail "CHECK PHƯƠNG ÁN", hỏi báo giá, xin tư vấn. Không có file cấp đơn thực sự — không cần ghi vào Excel.
🟡 Chuyển vào Needs_Review để bạn xem tay
TYPE D
Báo cáo / Thông báo nội bộ
Mail báo cáo doanh số, thông báo hệ thống, bản tin định kỳ. Không liên quan đến cấp đơn.
⏭️ Tự động bỏ qua, chuyển vào Skipped
REVIEW
Không xác định / Cần kiểm tra
Mail không rõ loại, có thể là cấp đơn bất thường. Hệ thống không chắc chắn nên để bạn kiểm tra tay.
🟡 Chuyển vào Needs_Review
ℹ️

Mail forward từ chị Linh (linh.nguyen2@affina.com.vn): Hệ thống đọc từ người gửi của bạn (chị Linh), không phải người gửi gốc bên trong phần trích dẫn. Miễn là email chị Linh được thêm vào SENDER_ALLOWLIST và mail forward có kèm file Excel/PDF hợp lệ, hệ thống sẽ nhận diện đúng TYPE và xử lý bình thường.

9. Folder Outlook sau khi pipeline chạy
Mail được chuyển vào đúng folder tương ứng

Sau khi xử lý, hệ thống tự chuyển mail vào các sub-folder trong Inbox → AutoFill. Cấu trúc folder:

📂
AutoFill/Incoming
Folder bạn cần forward mail vào (nếu dùng Outlook rule). Hệ thống sẽ đọc từ đây thay vì Inbox chính.
✅
AutoFill/Processed/YYYY-MM-DD
Mail TYPE A/B đã xử lý thành công và ghi vào Excel tổng. Được tổ chức theo ngày.
🔍
AutoFill/Needs_Review
Mail TYPE C (check phương án), mail không xác định, hoặc mail TYPE A/B nhưng validate bị thiếu field bắt buộc. Bạn cần xem và xử lý tay các mail trong này.
⏭️
AutoFill/Skipped/YYYY-MM-DD
Mail TYPE D (báo cáo, thông báo nội bộ) — hệ thống đã xem và xác định không cần xử lý.
📌

Lần đầu chạy, hệ thống tự tạo các folder này trong Outlook. Nếu muốn tạo thủ công ngay, chạy lệnh: python -m auto_fill outlook setup-folders

⚠️

Hàng ngày: Kiểm tra folder Needs_Review — đây là nơi chứa các mail mà hệ thống không xử lý được tự động. Mỗi sáng nên xem qua để không bỏ sót hồ sơ quan trọng.

11. Câu hỏi thường gặp
Các tình huống hay gặp khi sử dụng
Mail chị Linh forward cho tôi có cấp đơn — hệ thống xử lý được không?
Được, miễn là email linh.nguyen2@affina.com.vn đã được thêm vào SENDER_ALLOWLIST trong file .env. Hệ thống nhìn vào người gửi trực tiếp (chị Linh), không nhìn vào nội dung trích dẫn bên trong. File đính kèm và tiêu đề mail forward của chị Linh sẽ được dùng để phân loại.
Mail chị Linh forward có hồ sơ trùng với dữ liệu cũ trong Excel tổng — xử lý thế nào?
Hệ thống tự phát hiện và bỏ qua hồ sơ trùng — không ghi đè, không xóa dữ liệu cũ. Log sẽ hiện [dedup] Bỏ qua: <tên/CCCD>. Mail vẫn được chuyển vào folder Processed. Nếu hồ sơ cần được cập nhật (ví dụ phí thay đổi), bạn cần sửa tay trực tiếp trong file Excel tổng.
Folder Needs_Review trong Outlook là gì? Khi nào tôi cần vào đó?
Needs_Review chứa các mail mà hệ thống không tự xử lý được: mail "CHECK PHƯƠNG ÁN", mail thiếu field bắt buộc, hoặc mail không rõ loại. Kiểm tra mỗi sáng — nếu có hồ sơ quan trọng trong này, bạn cần nhập tay vào Excel tổng.
Pipeline chạy xong nhưng số vẫn là 0?
Có thể: (1) không có mail mới phù hợp trong Inbox, (2) email người gửi chưa trong allowlist, (3) tất cả mail đều là TYPE C/D và bị bỏ qua. Xem log để biết chính xác — log sẽ in ra TYPE của từng mail.
Mail đã bị chuyển sang Processed nhưng không thấy trong Excel tổng?
Kiểm tra Log Console — có thể hồ sơ bị trùng lặp (dedup), lỗi validation (thiếu trường bắt buộc), hoặc là TYPE C/D đã bị bỏ qua. Log sẽ ghi rõ lý do từng hồ sơ.
Nhấn "Chạy ngay" nhưng badge vẫn ở "Chờ" không chuyển?
Kiểm tra Terminal xem server có đang chạy không. Nếu Terminal bị đóng, server đã tắt — chạy lại python -m auto_fill serve rồi thử lại.
Muốn chạy lại mail đã bị chuyển sang Processed?
Mở Outlook → vào folder Processed → drag mail trở về Inbox (hoặc AutoFill/Incoming) → chạy pipeline lại. Tuy nhiên nếu hồ sơ đã ghi vào Excel tổng, lần chạy lại sẽ bị dedup bỏ qua.
Lỡ tay chạy pipeline 2 lần, có bị ghi trùng không?
Không — hệ thống có 2 lớp bảo vệ: (1) dedup theo CCCD + ngày + nơi đến, (2) tracker mail ID đã xử lý. Hồ sơ và mail đã xử lý sẽ bị bỏ qua hoàn toàn ở lần chạy thứ 2.
File Excel tổng trên Dashboard hiển thị dữ liệu cũ?
Nhấn nút 🔄 Làm mới trong phần "📊 File Excel Tổng" để tải lại từ Google Drive ngay lập tức (bỏ qua cache 5 phút).
Dashboard không hiển thị đúng trên màn hình nhỏ / cửa sổ nhỏ?
Từ v0.7.0, Dashboard đã được tối ưu responsive. Trên màn hình nhỏ, menu thu gọn thành nút ☰. Nếu vẫn có vấn đề, thử Ctrl+Shift+R để reload lại page.
Cần backup file tổng, làm sao?
Hệ thống tự backup vào data/backup/YYYY-MM-DD/ trước mỗi lần ghi. Giữ 30 ngày gần nhất. Nếu cần rollback: python -m auto_fill rollback --to data/backup/2026-05-14/master.xlsx
Muốn thêm loại bảo hiểm mới (ngoài 6 loại hiện tại)?
Báo cho developer biết tên sheet và cấu trúc file đính kèm — sẽ thêm vào hệ thống. Hiện tại hỗ trợ: Du lịch, Sức khỏe, Ô tô, Xe máy, BHYT/BHXH, HSSV.
